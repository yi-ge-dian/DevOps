执行安装脚本

每个节点都安装下 mysqlsh，记得取消mgr的注释

```shell
cd /usr/local
wget https://cdn.mysql.com//Downloads/MySQL-Shell/mysql-shell-8.0.46-linux-glibc2.17-x86-64bit.tar.gz
tar xvf mysql-shell-8.0.46-linux-glibc2.17-x86-64bit.tar.gz
ln -s mysql-shell-8.0.46-linux-glibc2.17-x86-64bit mysqlsh
cat >> /etc/profile << 'EOF'
export PATH=/usr/local/mysqlsh/bin:$PATH
EOF
source /etc/profile
mysqlsh -V
```

首先，用管理员账号 root 连接到第一个节点：
```shell
#在本地通过socket方式登入
mysqlsh -S /data/3306/run/mysql.sock -u root -p
```

执行命令 \status 查看当前节点的状态，确认连接正常可用。
执行 `dba.configureInstance()` 命令开始检查当前实例是否满足安装MGR集群的条件，如果不满足可以直接配置成为MGR集群的一个节点：
```shell
> dba.configureInstance()
2
mgr
123456
123456
```
接下来，用同样方法先用 root 账号分别登入到另外两个节点，完成节点的检查并创建最小权限级别用户（此过程略过。。。注意各节点上创建的用户名、密码都要一致），

完成检查并创建完新用户后，退出当前的管理员账户，并用新创建的MGR专用账户登入，准备初始化创建一个新集群：

```shell
mysqlsh --uri mgr@10.0.0.51:3306
> var c = dba.createCluster('MGR01');
```

执行 addInstance() 添加另外两个节点，这就完成了MGR集群的初始化并加入第一个节点（引导节点）。

```shell
> var c = dba.getCluster();
> c.addInstance('mgr@10.0.0.52:3306');
> c.addInstance('mgr@10.0.0.53:3306')
> c.status()

# 如果三台节点都关机后重启
> dba.rebootClusterFromCompleteOutage('MGR01');
```
