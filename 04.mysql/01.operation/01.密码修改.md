没有密码可以设置密码

```shell
# 方式一：利用命令操作
mysqladmin -uroot password 'oldboy123'

# 方式二：利用SQL语句操作
mysql 
alter user root@'localhost' identified by '123456';
```
已有密码可以修改密码
```shell
# 方式一：利用命令操作
mysqladmin -uroot -poldboy123 password '123456'

# 方式二：利用SQL语句操作
mysql -uroot -poldboy123
alter user root@'localhost' identified by '123456';
```

已有密码但是忘记密码（重置密码）
```shell
客户端发送连接请求：携带用户名、密码、目标地址和端口等信息，以数据包形式发送给服务端。
服务端接收请求：检测客户端的地址和端口是否允许连接。
调用验证模块：服务端加载授权表 mysql.user 进行身份核对。
- 授权表存储在磁盘中，验证时加载到内存中进行比对。
验证结果：
- 验证通过：允许客户端登录访问数据库。
- 验证失败：拒绝连接请求，返回错误信息。
# 步骤一：关闭数据库服务
mysql> shutdown;

# 步骤二：采用安全模式启动数据库（可以免密登录 -- 不会加载授权表）
/usr/local/mysql/bin/mysqld_safe --datadir=/data/3306/data --skip-grant-tables --skip-networking &

# 步骤三：重置密码信息
flush privileges;
# 作用一：可以将内存中授权表信息-- 同步写入磁盘  授权/用户数据 内存 -- 磁盘
# 作用二：可以将磁盘中授权表信息-- 加载到内存中  授权/用户数据 磁盘 -- 内存
alter user root@'localhost' identified by '123123';  --- mysql.user
ERROR 1290 (HY000): The MySQL server is running with the --skip-grant-tables option so it cannot execute this statement

# 步骤四：重新启动数据库服务
pkill mysql
/etc/init.d/mysqld start

# 步骤五：进行登录访问测试
mysql -uroot -p123123
```
