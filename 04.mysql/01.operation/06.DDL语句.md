# 1. 数据库操作

## 1.1. 创建数据库

```SQL
-- 最简单的创建
CREATE DATABASE oldboy;

-- 指定字符集创建
CREATE DATABASE oldboy CHARACTER SET utf8mb4;

-- 指定字符集和校对规则
CREATE DATABASE mydatabase CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;

CREATE DATABASE mydatabase CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 创建敏感字符名称的数据库（使用反引号,一般换一个）
CREATE DATABASE `select`;
```

## 1.2. 查看数据库

```SQL
-- 查看所有数据库
SHOW DATABASES;

-- 模糊匹配查询
SHOW DATABASES LIKE '%old%';

-- 查看数据库创建语句
SHOW CREATE DATABASE oldboy;
```

## 1.3. 修改数据库

修改字符集/校对规则

```SQL
-- 修改字符集
ALTER DATABASE test CHARSET utf8mb4;

-- 修改字符集和校对规则
ALTER DATABASE test CHARSET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

## 1.4. 删除数据库

```SQL
-- 删除数据库（生产环境需谨慎）
DROP DATABASE test;
DROP SCHEMA test;  -- 与上一条等价
```

## 1.5. 切换数据库

```SQL
-- 切换当前数据库
USE test;

-- 查看当前所在数据库
SELECT DATABASE();
```

# 2. 数据表操作

## 2.1. 创建数据表

```SQL
create table <表名> (
        <字段名1> <类型1> 字段部分约束 字段属性,
        …
        <字段名n> <类型n> 字段部分约束 字段属性,
        字段索引设置 （主键索引 辅助索引 联合索引）
        ) 引擎设置 字符集设置 校对规则设置;
  

CREATE TABLE `student` (
  `id` INT NOT NULL COMMENT '学号',
  `name` VARCHAR(45) NOT NULL COMMENT '姓名',
  `age` TINYINT UNSIGNED NOT NULL COMMENT '年龄',
  `gender` ENUM('M','F','N') NOT NULL DEFAULT 'N' COMMENT '性别',
  PRIMARY KEY (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='学生表';
```

## 2.2. 查看数据表

```SQL
-- 查看当前库所有表
SHOW TABLES;

-- 模糊查询表是否存在
SHOW TABLES LIKE '%student%';

-- 查看表的创建语句（可看到注释）
SHOW CREATE TABLE student;

-- 查看表结构（字段、类型、约束等）
DESC student;

-- 查看表索引信息
SHOW INDEX FROM student;
```

## 2.3. 修改数据表

```SQL
-- 修改表名
RENAME TABLE stu1 TO stu2;
ALTER TABLE stu2 RENAME TO stu3;

-- 修改表字符集
ALTER TABLE stu1 CHARSET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 添加字段

-- 1. 添加字段到最后
ALTER TABLE t1 ADD COLUMN telno CHAR(11) NOT NULL UNIQUE KEY COMMENT '手机号';

-- 2. 添加到指定字段之后
ALTER TABLE t1 ADD COLUMN wechat VARCHAR(64) NOT NULL UNIQUE KEY COMMENT '微信号' AFTER name;

-- 3. 添加到最前面
ALTER TABLE t1 ADD COLUMN qqnum VARCHAR(64) NOT NULL UNIQUE KEY COMMENT 'QQ号' FIRST;
-- PS：添加字段信息时，如果表中有数据信息，可能添加的字段信息会有冲突报错 

-- 删除字段
ALTER TABLE t1 DROP COLUMN qqnum;

-- 修改字段名称
-- 修改字段名和类型
ALTER TABLE t1 CHANGE old_name new_name VARCHAR(50) NOT NULL;

-- 仅修改字段类型
ALTER TABLE t1 CHANGE COLUMN `id` `id` BIGINT NOT NULL AUTO_INCREMENT ;
ALTER TABLE t1 CHANGE COLUMN `id` `id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT ;
alter table t1 modify name varchar(64) not null unique comment '学生名称';
```

## 2.4. 删除数据表

```SQL
-- 删除表（谨慎操作）
DROP TABLE t1;

delete/truncate  t1; -- 清空表中数据
delete from t1;
truncate table t1;
```

各种删除的区别

| **区别分析** | **drop table** **stu****;** | **truncate table** **stu****;** | **delete from** **stu****;** |
| ------------ | --------------------------- | ------------------------------- | ---------------------------- |
| 功能效果     | 删除表结构 + 数据           | 删除表数据（释放空间）          | 删除表数据（标记删除）       |
| 删除逻辑     | 彻底删除                    | 物理删除（段区页面删除）        | 逻辑删除（逐行删除）         |
| 删除效率     | 效率快（和数据量无关）      | 效率快（和数据量无关）          | 效率慢（和数据量有关）       |
| 自增影响     | 新增自增序列                | 重置自增序列（释放高水位线）    | 延续自增序列                 |
| 数据恢复     | 利用日志文件恢复            | 利用备份恢复/延时从库恢复       | 利用日志文件恢复（快速）     |

# 3. DDL 规范说明

- 创建数据表名称规范：要和业务有关（含库前缀），不要有大写字母，不要数字开头，不要含有系统关键字信息，名称不要太长； 
- 创建数据表属性规范：属性信息显示设置，引擎选择 InnoDB，字符集选择 utf8/utf8mb4，表信息添加注释； 
- 创建数据列属性规范：名称要有意义，不要含有系统关键字信息，名称不要太长； 
- 创建数据类型的规范：数据类型选择合适的、足够的、简短的； 
- 创建数据约束的规范：每个表中必须都要有主键，最好是和业务⽆关列，实现自增功能，建议每个列都非空（避免索引失效），加注释 
- 删除数据表操作规范：对于普通⽤户不能具有删表操作，需要操作删表时需要严格审核 
- 修改数据表结构规范：在数据库 8.0 之前，修改数据表结构需要在业务不繁忙时进行，否则会产生严重的锁
