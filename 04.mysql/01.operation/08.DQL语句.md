# 2. 查询获取服务数据信息【单表查询】

## 2.1. 基础查询数据方法：select + from + where

### 2.1.1. 全表信息查看

```SQL
select * from 表名;
select 字段01,字段02 from 表名;
```

### 2.1.2. 信息过滤查看

方式一：等值条件信息进行数据查询

- 查询中国的所有城市信息，中国代码信息 ”CHN”

```SQL
select * from city where CountryCode='CHN';
```

方式二：区间条件信息进行数据查询  `< > <= >= != <>` 

- 查询大于700万人的所有城市信息

```SQL
SELECT * FROM city WHERE population>7000000;
```

- 查询小于等于1000人的所有城市信息

```SQL
SELECT * FROM city WHERE population<=1000;
```

方式三：定义逻辑条件信息进行数据查询 and 并且 or 或者

- 查询中国境内，大于520万人口的城市信息

```SQL
SELECT * FROM city WHERE countrycode='CHN' AND population>5200000;
```

- 查询中国和美国的所有城市

```SQL
SELECT * FROM city WHERE countrycode='CHN' OR countrycode='USA';
```

- 查询人口数在100w到200w之间的城市信息

```SQL
SELECT * FROM city WHERE population>=1000000 and population<=2000000;
```

方式四：定义模糊条件信息进行数据查询（like）

- 查询国家代号是 CH 开头的城市信息

```SQL
SELECT * FROM city WHERE countrycode LIKE 'CH%';
```

- 查询国家代号含US内容的城市信息

```SQL
SELECT * FROM city WHERE countrycode LIKE '%US%';
```

方式五：特殊查询条件组合进行数据查询（配合in， not in， between and ）

- 查询中国和美国的所有城市

```SQL
SELECT * FROM city WHERE countrycode in ('CHN','USA');
SELECT * FROM city WHERE countrycode='CHN' OR countrycode='USA'
```

> in 的查询条件方式表示包含意思，实际应用更广泛

- 查询世界上的所有城市信息，但排除中国和美国的城市不查询

```SQL
SELECT * FROM city WHERE countrycode not in ('CHN','USA');
```

> not in 的查询条件方式表示排除意思，实际应用比较少见，因为not in不能走索引扫描，查询检索性能较慢

- 查询人口数量在50w-100w之间的城市信息

```SQL
SELECT * FROM city WHERE population>=500000 and population<=1000000;
SELECT * FROM city WHERE population between 500000 and 10000000;
```

> between and的查询条件方式是包含边界取值信息的，即包含50w人口的城市，也包含100w人口的城市

方式六：查询数据信息为空的内容（is null）

- 查询国家编码字段为空的信息

```SQL
select * from city where CountryCode is null;
```

- 查询国家编码字段为非空的信息

```SQL
select * from city where CountryCode not null;
```

## 2.2. 进阶查询数据方法：select + from + where + group by + having + order by + limit

### 2.2.1. 分组查询数据：group by

分组逻辑：

- 将相同的数据进行排序在一起  order by
- 将相同行的数据进行取重合并  distinct

分组输出需求：

当使用分组指令后，将信息输出时，必须确保输出的信息是一对一的关系输出，利用聚合函数

- 查询统计每个国家的人口总数

```SQL
select countrycode,sum(Population) from city group by countrycode;
```

- 查询统计中国每个省份的城市个数

统计/分析 函数信息 分组应用的函数信息 通常也称为聚合函数

- sum()    进行求和运算
- count()  进行数值统计
- avg()    进行取平均值
- max()    取出最大值
- min()    取出最小值

```SQL
select District,count(name) from city where countrycode='CHN'  group by District;
+----------------+-------------+
| District       | count(name) |
+----------------+-------------+
| Shanghai       |           1 |
| Peking         |           2 |
| Chongqing      |           1 |
| Tianjin        |           1 |
| Hubei          |          22 |
| Heilongjiang   |          21 |
| Liaoning       |          21 |
| Guangdong      |          20 |
```

- 查询统计中国每个省份的城市个数,以及输出每个省份城市名
   - group_concat()  可以将多行信息聚合成一行显示，多行内容用逗号分割

```SQL
select District,count(name),group_concat(name) from city where countrycode='CHN'  group by District;
```

### 2.2.2. 分组过滤查询：having

> having 与 where 的区别
>
> where  根据表的列信息进行过滤    过滤信息在表中存在
>
> having 根据聚合后的数据进行过滤  过滤信息在表中不存在

- 查询统计每个国家的人口总数，只显示人口数量超过1个亿的信息

```SQL
select countrycode,sum(Population) from city group by countrycode having sum(Population)>100000000;
```

### 2.2.3. 排序查询数据：order by

可以将输出信息，根据指定列进行排序显示

- 查询统计中国每个省份的城市个数，并且根据城市个数进行排序显示

```SQL
select District,count(name) from city where countrycode='CHN'  group by District order by count(name);      -- 升序
select District,count(name) from city where countrycode='CHN'  group by District order by count(name) desc; -- 降序
```

- 查询统计中国国家的每个省份人口总数，只显示人口数量超过800w的信息，并按照人口数量排序显示

```SQL
select District,sum(Population) from city 
where countrycode='CHN' group by District 
having sum(Population)>8000000 
order by sum(Population) desc;

+--------------+-----------------+
| District     | sum(Population) |
+--------------+-----------------+

| Shandong     |        12114416 |

| Jiangsu      |         9719860 |
| Shanghai     |         9696300 |
| Guangdong    |         9510263 |
| Hubei        |         8547585 |
+--------------+-----------------+
```

### 2.2.4. 截取信息输出：limit

- 查询统计每个国家的人口总数，只显示人口数量超过5千万的信息，并且按照国家人口总数从大到小排序，只显示前三名

```SQL
select countrycode,sum(Population) from city group by countrycode having sum(Population)>50000000 order by sum(Population) desc limit 3;
select countrycode,sum(Population) from city group by countrycode having sum(Population)>50000000 order by sum(Population) desc limit 0,3;
```

> limit N,M : N表示截取信息起始点，M表示截取信息数据行数

```SQL
select countrycode,sum(population) from world.city group by countrycode having sum(population)>50000000 order by sum(population) desc limit 3 offset 0;
```

> limit M offset N : N表示截取信息起始点，M表示截取信息数据行数

- 查询统计每个国家的人口总数，只显示人口数量超过5千万的信息，并且按照国家人口总数从大到小排序，只显示三~五名

```SQL
select countrycode,sum(Population) from city group by countrycode having sum(Population)>50000000 order by sum(Population) desc limit 2,3;
select countrycode,sum(population) from world.city group by countrycode having sum(population)>50000000 order by sum(population) desc limit 3 offset 2;
```

## 3.2. 纵向拼接

可以利用 union 指令实现将多个 select 命令输出的结合集合信息进行组合；

实现纵向拼接条件：

- 拼接的结果集的列数必须一致；

若不满足条件1，会输出的错误信息

```SQL
mysql> select name,age from t1 union select name from t2;
ERROR 1222 (21000): The used SELECT statements have a different number of columns
```

- 拼接列的数据类型和约束属性信息一致;

union / union all 区别：

union 会实现取消重复信息拼接

```SQL
mysql> select * from t1 union select * from t2;
+------+---------+------+
| id   | name    | age  |
+------+---------+------+
|    1 | xiaoA   |   18 |
|    2 | xiaoB   |   19 |
|    3 | xiaoQ   |   20 |
|    1 | oldboy  |   30 |
|    2 | oldgril |   31 |
+------+---------+------+
```

union all 拼接后信息不会取重，全部显示

```SQL
mysql> select * from t1 union all select * from t2;

+------+---------+------+
```

# 4. 查询获取服务数据信息【嵌套查询】

将一个查询语句完全嵌套到另一个查询语句中的的WHERE或HAVING的条件表达式中，这种查询称为嵌套查询；

通常把内部的、被另一个查询语句调用的查询称为"子查询"；

将调用子查询的查询语句称为"父查询"，子查询还可以可以调用子查询；

嵌套查询示例：

查询全球所有国家中，城市人口数量大于中国北京的城市信息；

```SQL
select * from city where Population>(select Population from city where countrycode='CHN' and name='peking');
```

## 4.1. 不相关子查询

一条 SQL 语句含有多个 select，先执行子查询，再执行外查询；子查询可以独立运行，就称为不相关子查询；

根据子查询的结果行数，可以分为单行子查询和多行子查询；

### 4.1.1. 单行子查询

单行子查询就是子查询输出的结果集为一行信息，可以直接进行比较即可；（<  > =）

- 查询工资高于平均工资的雇员名字和工资；

```SQL
select ENAME,SAL from EMP where SAL>(select avg(SAL) from EMP);
+-------+------+
| ENAME | SAL  |
+-------+------+
| JONES | 2975 |
| BLAKE | 2850 |
| CLARK | 2450 |
| SCOTT | 3000 |
| KING  | 5000 |
| FORD  | 3000 |
+-------+------+
```

- 查询和CLARK同一部门且比他工资低的雇员名字和工资；

```SQL
select ENAME,SAL from EMP where DEPTNO=(select DEPTNO from EMP where ENAME='CLARK') and SAL<(select SAL from EMP where ENAME='CLARK');
```

### 4.1.2. 多行子查询

多行子查询就是子查询输出的结果集为多行信息；

多行子查询就是子查询输出的结果集为多行信息，不能直接进行比较处理，需要借助 `in 或者 any 或者 all` 进行处理；

- 查询工资低于任意一个'CLERK'职位工资的雇员信息；

```SQL
select * from EMP where sal < (select max(sal) from EMP where job ='CLERK'); -- 处理问题方法
select * from EMP where sal < any(select sal from EMP where job ='CLERK');    
```

- 查询工资比所有的'SALESMAN'都高的雇员的编号、名字和工资

```SQL
select * from EMP  where sal > (select max(sal) from EMP where job ='SALESMAN');
select * from EMP where sal > all(select sal from EMP where job='SALESMAN');
```

- 查询部门20中职务同部门10的雇员一样的雇员

```SQL
select * from EMP where deptno = 20 and job in (select job from EMP where deptno=10);
select * from EMP where deptno = 20 and job = any (select job from EMP where deptno = 10);
```

## 4.2. 相关子查询

一条SQL语句含有多个select，先执行外查询，再执行子查询；子查询不可以独立运行，就称为相关子查询；

- 相关子查询优势：操作简单，功能强大（一些使用不相关子查询不能实现或者实现繁琐的子查询，可以使用相关子查询实现）
- 相关子查询缺陷：语句操作稍难理解

### 4.2.1. 表别名

```SQL
select teacher.tname,group_concat(student.sname)
from teacher
join course
on teacher.tno=course.tno
join sc
on course.cno=sc.cno
join student
on sc.sno=student.sno
where sc.score<60
group by teacher.tno;

--> 

select a.tname,group_concat(d.sname)
from teacher a
join course  b
on a.tno=b.tno
join sc c
on b.cno=c.cno
join student  d
on c.sno=d.sno
where c.score<60
group by a.tno;
```

### 4.2.2. 列别名

```SQL
select a.tname 讲师名称,group_concat(d.sname) 不及格学生信息
from teacher as a
join course as b
on a.tno=b.tno
join sc as c
on b.cno=c.cno
join student as d
on c.sno=d.sno
where c.score<60
group by a.tno;
```

# 4. 总结

利用select命令获取表中数据方法

- 单表简单数据信息获取： select + from + where

应用：单表已有数据信息过滤

- 单表进阶数据信息获取： select + from + where + group by + having + order by + limit

应用：单表数据信息分析统计

- 多表拼接查询数据信息：

横向拼接：内连接 外连接

应用：查询的数据信息分散在多个不同业务表中

纵向拼接：union

应用：相同表数据信息比较或者汇总

- 嵌套查询数据信息

应用：开发程序代码调取数据库数据信息时（看的懂）
