# 1. 插入数据（INSERT）

```SQL
-- 标准语法
insert into 表名 (字段01名称, 字段02名称, ...) values (字段01值, '字段02值', ...);
-- 精简语法
insert into 表名 (必要字段信息) values (必要字段值);
```

**表结构说明**

- `id`：自增主键列，非必要字段
- `age`：有默认值，非必要字段
- `telnum`：可以为空，非必要字段

```SQL
-- 插入单条记录
insert into 表名 values (1, 'xiaoQ', 18);

-- 插入多条记录
insert into 表名 values (1, 'xiaoQ', 18), (2, 'xiaoR', 20);
```

# 2. 修改数据（UPDATE）

标准语法：

```SQL
update 表名 set 字段名 = '字段值' where 过滤行信息;
```

示例：

```SQL
-- 查看满足条件的数据
select * from t1 where name = 'xiaoA' and telno is null;
select * from t1 where id = 3;

-- 更新数据
update t1 set telno = '444' where id = 2;
```

# 3. 删除数据

标准语法：

```SQL
delete from 表名 where 过滤行信息;
```

示例：

```SQL
delete from t1 where telno = '111';
```

## 3.1. 如何防止 update/delete 误操作？

第一个方法：实现表数据伪删除，加一列 flag

第二个方法：禁止不加where执行语句

sql_safe_updates  -- 避免delete或update出现误操作

官方资料参考：https://dev.mysql.com/doc/refman/8.4/en/server-system-variables.html

**服务端配置**

- 临时配置

```SQL
set global 配置项名称/变量名称='';     -- 只要数据库服务不重启，配置信息会一直生效，并且会影响所有管理用户
set [session] 配置项名称/变量名称='';  -- 会话配置信息只在当前会话生效

-- 开启安全更新模式（全局级别）
mysq l> set global sql_safe_updates = 1;
-- 注意：需要退出当前会话重新登录才能生效

-- 执行无条件更新操作
mysql> update stu set sname = 'oldboy';
-- 报错信息如下：
ERROR 1175 (HY000): You are using safe update mode and you tried to update a table without a WHERE that uses a KEY column.
```

- 永久生效（修改配置文件）

```Bash
# 创建脚本文件，写入开启安全更新的命令
echo 'set global sql_safe_updates = 1;' > /opt/init.sql
# 赋予执行权限
chmod +x /opt/init.sql


vi /etc/my.cnf
[mysqld]
init-file = /opt/init.sql

systemctl restart mysqld
mysql -uroot -poldboy123 -e "select @@global.sql_safe_updates"
+-----------------------------+
| @@global.sql_safe_updates   |
+-----------------------------+
|                           1 |
+-----------------------------+
```

**客户端配置**

```SQL
-- 刚到xx公司管理数据库 10.0.0.51 3306 
-- centos系统 -- 部署二进制8.0 -- mysql命令 -- /etc/my.cnf
mysql -uxxx  -pxxx  -h10.0.0.51 -P3306  -- update / delete 可以进行保护了
vim /etc/my.cnf
[mysql]
safe_updates=1
   
alias mysql='mysql -U'
```
