查看权限:
```shell
# 展示所有的权限
show priviledges;

# 展示某个用户的权限
show grants for 用户名@'主机白名单'
```

常用的表权限：

| **动作**       | **对象**                              | **能力**                                                     |
| -------------- | ------------------------------------- | ------------------------------------------------------------ |
| Alter          | Tables(表)                            | To alter the table （修改表属性能力）                        |
| Create         | Databases,Tables,Indexes              | To create new databases and tables （创建库 创建表（创建索引）） |
| Create user    | Server Admin                          | To create new users （创建用户能力）                         |
| Drop           | Databases,Tables                      | To drop databases, tables, and views （删除库 删除表 删除视图） |
| Index          | Tables                                | To create or drop indexes （创建和删除索引能力）             |
| Show databases | Server Admin                          | To see all databases with SHOW DATABASES （查看所有数据库信息能力） |
| Grant option   | Databases,Tables,Functions,Procedures | To give to other users those privileges you possess （是否有给予玉玺能力） |
| Usage          | Server Admin                          | No privileges - allow connect only （只允许用户登录数据库权限） |
| Insert         | Tables                                | To insert data into tables （添加表中行）                    |
| Delete         | Tables                                | To delete existing rows （删除表中行）                       |
| Select         | Tables                                | To retrieve rows from table（查看表中行）                    |
| Update         | Tables                                | To update existing rows （修改表中行）                       |

设置权限

在 MySQL 中，权限的分配主要通过 GRANT语句实现，但在不同版本中存在一定的语法与行为差异，需特别注意。

MySQL 8.0 授权方式。

```shell
GRANT 权限类型 ON 权限对象 TO 用户角色;
```

注意：执行此授权前，目标用户必须已存在，否则会报错。权限设置与用户创建是两个独立操作。
MySQL 5.6 / 5.7 授权方式。

```shell
GRANT 权限类型 ON 权限对象 TO 用户角色 IDENTIFIED BY '密码';
```
注意：此语句在授予权限的同时，会自动创建指定的用户（若用户不存在），并设置对应密码。这是与 8.0 版本的重要区别之一。

```shell
create user xiaoA@'%' identified by '123456';
create database xiaoA;
# 授予创建 xiaoA 库创建表、索引的权限
grant create on xiaoA.* to xiaoA@'%';
# 增加权限
grant select,insert,delete,update on xiaoA.* to 'xiaoA'@'%';
# 回收权限
revoke delete on xiaoA.* from 'xiaoA'@'%';
```

特殊权限一：ALL /ALL PRIVILEGES
```shell
grant all on *.* to xiaoQ@'10.0.0.%';
```
特殊权限二：grant option
```shell
grant Grant option on *.* to xiaoQ@'10.0.0.%'; 

grant select,update on *.* to xiaoQ@'10.0.0.%' with Grant option;
```

数据库服务中和权限相关的授权表有什么?

授权表如下：
- user，记录全局权限设置信息
- db  ，记录指定数据库权限信息
- tables_priv，记录指定数据表权限信息

创建三个用户
```sql
use xiaoA;
create table t1 (id int);
create user xiaoA@'%' identified by '123456';
create user xiaoB@'%' identified by '123456';
create user xiaoC@'%' identified by '123456';
```

授予权限
```sql
-- user    可以看到权限对象*.* 的全局权限信息 
grant select,update,insert on *.* to xiaoA@'%' ;

-- db      可以看到局部权限设定信息 针对指定库
grant select,update,insert on xiaoA.* to xiaoB@'%' ;

-- tables_priv  可以看到局部权限设定信息 针对指定表
grant select,update,insert on xiaoA.t1 to xiaoC@'%' ; 

show grants for 用户名@'主机白名单'
```

数据库 8.0 版本之后，用户权限设置新功能 -- 角色设置
当 mysql 数据库中，具有多个管理用户，并且多个管理用户权限设置比较统一时，如果想进行统一批量管理，就可以应用角色功能。
```shell
create database oldboy;
use oldboy;
create table t1 (id int);
insert into t1 values (1),(2),(3);
select * from oldboy.t1;

-- 步骤一：创建角色信息
create role oldboy_rw,oldboy_r;          

-- 步骤二：给角色信息设置权限
grant select on oldboy.* to oldboy_r; -- 只读角色权限设置
grant select,update,insert,delete on oldboy.* to oldboy_rw; -- 读写角色权限设置 

-- 步骤三：创建用户信息，并且将用户信息划分到指定角色中  
create user user01@'%' identified by '123';
create user user02@'%' identified by '123';
grant oldboy_r to user01@'%'; 
grant oldboy_rw to user02@'%';           

-- 步骤四：激活角色功能
select current_role();
-- 查看角色功能是否激活命令
    -- 没有激活时，会输出显示none
    -- 已经激活后，会显示角色信息。

-- 方式一：手工激活
set default role all to user01@'%';
-- 方式二：自动激活
set global activate_all_roles_on_login=on;  -- 管理员登录数据库进行设置
mysql> select @@activate_all_roles_on_login;
+-------------------------------+
| @@activate_all_roles_on_login |
+-------------------------------+
|                             1 |
+-------------------------------+
```
