创建用户：
在数据库管理中，用户的创建遵循  '用户名'@'主机白名单' 的格式，其核心组成与规则如下：
- 用户名：用于标识用户身份，是登录时的凭据之一。
- 主机白名单：用于限制该用户允许连接的主机来源，支持具体 IP、网段或通配符表示。
```shell
create user 用户名@'主机白名单' identified by '密码';
```
常见的数据库连接方式主要包括两种：
本地登录
用户通过  localhost 或  127.0.0.1 从数据库所在服务器本地进行登录。
远程登录
用户从指定网络位置进行连接，支持以下常见主机白名单形式：
- 单个主机：如 10.0.0.52
- 域名形式：如 web01.example.com（需可解析）
- 网段形式：
  - 10.%（A 类私有地址段）
  - 172.16.%（B 类私有地址段）
  - 192.168.%（C 类私有地址段）
  - 10.0.0.%（等效于 CIDR 表示 10.0.0.0/24，企业内常见）
- 全部主机：%（允许来自任意远程主机的连接，需谨慎使用）


数据库创建用户后，无法远程登录，显示认证错误
```shell
./mysql -utest01 -p123 -h10.0.0.51
```
> Warning: Using a password on the command line interface can be insecure.
ERROR 2059 (HY000): Authentication plugin 'caching_sha2_password' cannot be loaded: /usr/local/mysql/lib/plugin/caching_sha2_password.so: cannot open shared object file: No such file or directory.

出现此问题原因：新版数据库密码加密插件进行了更新

解决问题方式一：升级客户端，使客户端加密密码方式支持caching_sha2_password

解决问题方式二：降级服务端，使服务端加密密码方式支持mysql_native_password
```shell
alter user my_user@'10.0.0.%' identified with mysql_native_password by '123456';
```

查询用户：
```shell
select user,host,plugin from mysql.user;
select user();
show processlist;
```

删除用户:
在生产环境中慎用，不使用的用户可以先锁定，并查看是否在线，及时沟通。
```shell
drop user 用户名@'主机白名单';

select user,host from mysql.user where user='xiaoA' or user='xiaoB' or user='xiaoQ';

delete from mysql.user where user='xiaoA' or user='xiaoB' or user='xiaoQ';
```

锁定与解锁用户
```shell
#锁定
alter user 用户名@'主机白名单' account lock;
#解锁
alter user 用户名@'主机白名单' account unlock;
#查看状态
select user,host,account_locked from mysql.user;
```
