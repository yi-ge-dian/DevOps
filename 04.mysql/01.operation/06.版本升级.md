# 1. 数据库版本升级方法

方案一：本地升级（In-place Upgrade）

- **部署方式**：在单台服务器上直接升级数据库。
- **操作描述**：将服务器 10.0.0.51 上的 MySQL 5.7 原地升级至 MySQL 8.0 并重新加载数据。
- **业务影响**：升级期间服务会中断，对线上业务有直接影响。

方案二：迁移升级（Migration Upgrade）

- **部署方式**：采用两台服务器，通过数据迁移完成升级。
- **操作描述**：
  - 保持 10.0.0.51 的 MySQL 5.7 数据正常提供。
  - 将数据迁移至新服务器 10.0.0.52 的 MySQL 8.0 并恢复。
  - 在 10.0.0.52 进行业务测试与验证。
  - 可配置主从同步，便于切换与回滚。
- **业务影响**：迁移过程中旧服务保持运行，业务影响最小，可实现平滑过渡。

# 2. 数据库服务升级规范

数据库服务升级官方参考资料：https://dev.mysql.com/doc/refman/8.0/en/upgrade-paths.html

- **版本支持原则**：仅支持在正式发布版本（GA）之间进行升级，不适用于开发或预览版。
- **升级路径规划**：支持从 MySQL 5.6 升级到 8.0 的跨大版本升级。标准路径为：`5.6 → 5.7 → 8.0`。操作时，需先将 5.6 升级至其最新的小版本，再执行后续的跨版本升级。
- **风险与回退预案**：升级前必须制定明确的版本回退方案。强烈建议在升级前，特别是升级至 8.0 版本时，执行完整的数据备份，这是保障数据安全和实现快速回退的基础。
- **效率与影响控制**：制定升级方案与详细操作步骤时，核心目标之一是最大限度地减少数据库服务的停机时间，以降低对业务的影响。

# 3. 数据库升级流程

数据备份

- 可采用热备（在线备份，不影响业务）或冷备（需暂停业务后进行）两种方案，确保升级前数据安全。

安装新版本程序

- 在服务器上安装最新版本的数据库程序。此过程无需停止正在运行的旧版本数据库服务。

切换与通知

- 关闭旧版本数据库服务，并将对外网站切换至维护页面。
- 提前发布升级通知，并在维护页面向用户清晰展示升级过程说明。

启动与数据升级

- 采用安全模式启动新数据库服务（使用 `--skip-grant-tables`和 `--skip-networking`参数），加载旧版本数据文件，完成基础的“挂库”操作。

升级系统数据结构

- 运行数据库升级工具，将系统表和数据字典升级至与新版本兼容的结构。此步骤耗时主要与系统复杂度相关，与业务数据量大小无关。

重启与基础验证

- 升级完成后，以正常模式重启数据库服务，并确认服务可稳定重启。

功能测试

- 进行全面、反复的功能测试验证，确保所有依赖数据库的应用程序功能正常。

恢复业务

- 撤销网站维护页面，恢复线上业务的正常运营，标志本次升级工作全部完成。

> 数据库升级涉及两个方面：
>
> 数据库程序升级：安装新的程序（安装新版数据库实例）    新的程序加载升级后的数据目录
>
> 数据库数据升级：数据目录结构升级/将授权表结构信息升级

## 4.2.环境准备

下载软件程序

```Bash
wget https://downloads.mysql.com/archives/get/p/23/file/mysql-5.7.40-linux-glibc2.12-x86_64.tar.gz
wget https://downloads.mysql.com/archives/get/p/23/file/mysql-5.6.48-linux-glibc2.12-x86_64.tar.gz
```

解压安装软件

```Bash
cd /usr/local/
tar xvf mysql-5.6.48-linux-glibc2.12-x86_64.tar.gz
ln -s mysql-5.6.48-linux-glibc2.12-x86_64  mysql56
# PS: 此时环境变量不要进行配置 

tar xvf mysql-5.7.40-linux-glibc2.12-x86_64.tar.gz
ln -s mysql-5.7.40-linux-glibc2.12-x86_64  mysql57  
# PS: 此时环境变量不要进行配置   
```

数据库软件初始化

```Bash
mkdir -p /data/3356/data
chown mysql.mysql  /data/3356/data
yum -y install perl-Data-Dumper
/usr/local/mysql56/scripts/mysql_install_db --user=mysql --datadir=/data/3356/data  --basedir=/usr/local/mysql56
# 输出信息中是否有两个OK
echo $?

mkdir -p /data/3357/data
chown mysql.mysql  /data/3357/data
/usr/local/mysql57/bin/mysqld --initialize-insecure --user=mysql --datadir=/data/3357/data  --basedir=/usr/local/mysql57
echo $?
```

编写数据库服务配置文件

```Bash
# 3356
cat > /data/3356/my.cnf <<eof
[mysqld]
user=mysql
port=3356
basedir=/usr/local/mysql56
datadir=/data/3356/data

# 3357
cat > /data/3357/my.cnf <<eof
[mysqld]
user=mysql
port=3357
basedir=/usr/local/mysql57
datadir=/data/3357/data
socket=/data/3357/mysql.sock
```

运行启动数据库服务

```Bash
# 3357
cat >/usr/lib/systemd/system/mysqld3357.service<<EOF
```


### 4.4.1. 提前测试能否升级成功

在从 MySQL 5.7 升级到 8.0 之前，可以使用 MySQL Shell 的 ` util.checkForServerUpgrade() `函数。此工具能够自动检查当前数据库的配置、数据兼容性等问题，并给出详尽的评估报告，以预先判断实例是否能安全升级到 8.0 版本，有效规避升级风险。

升级必须遵循官方支持的路径。例如，从 **5.6.20 版本升级到 8.0**，**禁止**直接跨版本升级。正确的路径是：**5.6.20 → 5.7.40（或5.7系列的最新版本） → 8.0**。必须先完成到中间版本（5.7）的升级，再升级至目标版本。

下载安装 mysql-shell 软件工具

https://downloads.mysql.com/archives/shell/

```Bash
cd /usr/local
# 下载需要升级到的版本
wget https://downloads.mysql.com/archives/get/p/43/file/mysql-shell-8.0.40-linux-glibc2.17-x86-64bit.tar.gz
tar xvf mysql-shell-8.0.40-linux-glibc2.17-x86-64bit.tar.gz
ln -s mysql-shell-8.0.40-linux-glibc2.17-x86-64bit mysqlsh

vim /etc/profile
export PATH=$PATH:/usr/local/mysqlsh/bin
source /etc/profile
```

进行检测低版本数据库程序

```Bash
cat /usr/lib/systemd/system/mysqld3357.service
[Unit]
Description=MySQL Server
After=network.target
After=syslog.target
 
[Install]
WantedBy=multi-user.target
 
[Service]
User=mysql
Group=mysql
ExecStart=/usr/local/mysql57/bin/mysqld --defaults-file=/data/3357/my.cnf
LimitNOFILE = 10000

cat /data/3357/my.cnf
[mysqld]
user=mysql
port=3357
basedir=/usr/local/mysql57
datadir=/data/3357/data
socket=/data/3357/mysql.sock

systemctl start mysqld3357
mysql -S /data/3357/mysql.sock
create user root@'10.0.0.%' identified by '123456';
grant all on *.* to root@'10.0.0.%';

mysqlsh root:123456@10.0.0.4:3357 -e "util.checkForServerUpgrade()"
# 执行命令，若没有error提示，表示可以实现版本升级 5.7.30 --> 8.0.26
```

### 4.4.2. 数据库回退流程

当数据库升级过程失败或升级后出现不可接受的严重问题时，需立即执行回退操作，以快速恢复服务。

关闭新版数据库服务程序

```Bash
systemctl stop mysqld3356
```

还原备份数据信息

如 5.6 升级到 5.7失败， 将备份 56 数据重新恢复到 56 程序的数据目录

```Bash
rm -rf /data/3356/*
cp -a /backup/3356/* /data/3356/
```

重新运行启动旧版本数据库服务

```Bash
# cat /data/3356/my.cnf 
[mysqld]
user=mysql
port=3356  # 是否有调整
basedir=/usr/local/mysql56  # 程序信息是否是旧版本程序
datadir=/data/3356/data     # 数据目录是否是旧版数据目录信息
socket=/data/3356/mysql.sock

vim /usr/lib/systemd/system/mysqld3356.service
[Unit]
Description=MySQL Server
After=network.target
After=syslog.target

[Install]
WantedBy=multi-user.target

[Service]
User=mysql
Group=mysql
ExecStart=/usr/local/mysql56/bin/mysqld --defaults-file=/data/3356/my.cnf
LimitNOFILE = 10000

systemctl daemon-reload
systemctl start mysqld3356
```

检查旧版数据库服务状态

```Bash
ps -ef|grep mysql
# mysql     12929      1  1 18:10 ?   00:00:00 /usr/local/mysql56/bin/mysqld --defaults-file=/data/3356/my.cnf
```
