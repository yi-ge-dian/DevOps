配置文件的加载顺序: `/etc/my.cnf -> /etc/mysql/my.cnf -> /usr/local/mysql/etc/my.cnf -> ~/.my.cnf`

如何编写配置文件内容，配置信息包含两个部分【客户端】与【服务端】：

| 段落标签      | 说明                     | 适用程序                               |
| ------------- | ------------------------ | -------------------------------------- |
| [client]      | 全局客户端配置           | 所有客户端工具（mysql、mysqladmin 等） |
| [mysql]       | mysql 客户端专属配置     | mysql命令                              |
| [mysqladmin]  | mysqladmin 专属配置      | mysqladmin命令                         |
| [mysqldump]   | mysqldump 专属配置       | mysqldump命令                          |
| [server]      | 全局服务端配置           | 服务端相关进程                         |
| [mysqld]      | mysqld 进程专属配置      | mysqld主进程                           |
| [mysqld_safe] | mysqld_safe 进程专属配置 | mysqld_safe守护进程                    |

查看配置项：

- 客户端：`mysql --help`、`mysqladmin --help`、`mysqldump --help`
- 服务端：`mysqld --help --verbose`（查看 `[mysqld]`配置项）
- mysqld_safe 相关配置可参考官方文档。

编写建议：

将通用设置放在 `[client]`或 `[server]`全局段，将程序特有的配置放在对应的专属段，便于管理与维护。
