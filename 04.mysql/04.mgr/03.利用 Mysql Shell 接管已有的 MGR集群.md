# 03.利用 Mysql Shell 接管已有的 MGR集群
https://gitee.com/GreatSQL/GreatSQL-Doc/blob/master/deep-dive-mgr/deep-dive-mgr-04.md

MySQL Shell 是一个客户端工具，可用于方便管理和操作 MySQL，支持 SQL、JavaScript、Python 等多种语言，也包括完善的 API。GreatSQL Shell 支持文档型和关系型数据库模式，通过 X DevAPI 可以管理文档型数据，通过AdminAPI 可以管理 InnoDB Cluster、InnoDB ClusterSet 及 InnoDB ReplicaSet 等。


安装 mysql-shell，在 51 master 节点安装即可
```shell
cd /usr/local
wget https://cdn.mysql.com//Downloads/MySQL-Shell/mysql-shell-8.0.46-linux-glibc2.17-x86-64bit.tar.gz
tar xvf mysql-shell-8.0.46-linux-glibc2.17-x86-64bit.tar.gz
ln -s mysql-shell-8.0.46-linux-glibc2.17-x86-64bit mysqlsh
cat >> /etc/profile << 'EOF'
export PATH=/usr/local/mysqlsh/bin:$PATH
EOF
source /etc/profile
mysqlsh -V
```

创建 root 用户，来进行远程连接（主节点安装一次，自动进行同步）
```sql
CREATE USER 'root'@'%' IDENTIFIED BY '123456';
GRANT ALL ON *.* TO 'root'@'%' WITH GRANT OPTION;
flush privileges;
SELECT user,host,plugin FROM mysql.user;
```

如果加上 adoptFromGr:true 选项，则会直接创建集群，不再询问：
```shell
var c=dba.createCluster('MGR01', {adoptFromGr:true});
```
如果是MGR集群的metadata发生变化，这时候无论调用 dba.getCluster() 还是 dba.createCluster 都可能会报告类似下面的错误：
```shell
Dba.getCluster: Unable to get an InnoDB cluster handle. The instance '10.0.0.51:3306' may belong to a different cluster from the one registered in the Metadata since the value of 'group_replication_group_name' does not match the one registered in the Metadata: possible split-brain scenario. Please retry while connected to another member of the cluster. (RuntimeError)
```
这种情况下，可以调用 dba.dropMetadataSchema() 函数删除元数据，再调用 dba.createCluster() 接管集群：
```shell
#确保不影响正常业务的话，删除无用MGR元数据
JS > dba.dropMetadataSchema()
Are you sure you want to remove the Metadata? [y/N]: y
Metadata Schema successfully removed.
#接管现有集群
JS > var c=dba.createCluster('MGR1', {adoptFromGr:true})
...
```
