# 1. 日志概述介绍

在数据库系统中，存在多种类型的日志，用于全面记录数据库运行的各个方面，帮助数据库管理员追踪和诊断数据库发生的各种事件。

这些日志主要针对数据库 **Server 层** 产生的数据信息，主要包括两类：

**数据库服务运行相关的日志：**主要用于记录数据库服务本身的运行状态、启动关闭信息、错误事件、性能指标等。

**SQL**语句操作执行相关的日志：主要用于记录用户执行的SQL语句、事务操作、数据修改等数据库操作信息。

# 2. 日志常用分类

日志信息查看⽅法：

```SQL
mysql> show variables like '%log%'; 
```

| **日志名称**   | **解释说明**                                                 |
| -------------- | ------------------------------------------------------------ |
| general_log    | 表示查询日志（通用日志），默认日志状态处于关闭，可以进行在线调整配置。作用：记录了客户端从会话连接开始，执行过的所有SQL语句信息； |
| log_error      | 表示错误日志（运行日志），默认日志状态处于激活。作用：记录了数据库服务启动和停止时，以及服务器在运行过程中发生任何严重错误时的相关信息； |
| log_bin        | 表示二进制日志（binlog日志），默认日志状态处于激活（8.0之后）。作用：记录了所有的DDL语句和DML语句，但是不包括数据库查询语句；语句以事件的形式保存，描述了数据的更改过程，此日志对于灾难时的数据恢复起着极其重要的作用。 |
| slow_query_log | 慢查询日志，记录了所有执行时间超过参数 long_query_time设置值并且扫描记录数小于 min_examined_row_limit的所有SQL语句的日志。 |

# 3. 日志信息配置

## 3.1. 通用日志

```SQL
mkdir -p /data/3306/log
chown -R mysql.mysql /data/3306/log
-- 创建日志信息存储路径

通用日志：日志相关配置

mysql> show variables like '%general_log%';
+------------------+----------------------------+
| Variable_name    | Value                      |
+------------------+----------------------------+
| general_log      | OFF                        |   -- 是否开启日志记录功能
| general_log_file | /data/3306/data/master.log |   -- 定义日志存储路径，以及日志名称
+------------------+----------------------------+
 
/etc/my.cnf 
general_log=on
general_log_file=/data/3306/log/general.log
```

日志功能作用：在进行业务访问数据库测试过程中，可以开启数据库服务通用日志；

## 3.2. 错误日志

```SQL
mysql> show variables like '%log_error%';
+----------------------------+----------------------------------------+
| Variable_name              | Value                                  |
+----------------------------+----------------------------------------+
| log_error                  | stderr                                 |
+----------------------------+----------------------------------------+

 
/etc/my.cnf 
log_error=/data/3306/log/xiaoQ_error.err
```

若配置错误日志文件后，重启数据库服务报错，无法创建错误日志，可以先手动创建好错误日志文件，在重启数据库服务

```SQL

| Log_name            | Pos | Event_type     | Server_id | End_log_pos | Info                              |
+---------------------+-----+----------------+-----------+-------------+-----------------------------------+
| xiaoQ_binlog.000006 |   4 | Format_desc    |      3306 |         125 | Server ver: 8.0.26, Binlog ver: 4 |
| xiaoQ_binlog.000006 | 125 | Previous_gtids |      3306 |         156 |                                   |
+---------------------+-----+----------------+-----------+-------------+-----------------------------------+
```

ddl 语句 ：创建库  创建表  ...  单个语句就是一个事务/事件

dml 语句 ：begin 添加  修改  删除  commit

>  sync_binlog  -- 记录binlog日志方式:
>
> -- 在进行主从同步过程的双一标准的其中一个1的信息配置，主要是控制缓冲区里的binlog日志信息如何刷写到磁盘中；
>
> -- 此参数信息是有三种方式进行配置的：
>
> -- 参数信息配置0：表示由操作系统缓存自己决定，什么时候刷新日志到磁盘中；
>
> -- 参数信息配置1：表示每次事务提交，立即刷新日志到磁盘中；(此方式配置更安全) 
>
> -- 参数信息配置N：表示每组事务提交，按照组的事务次数定义，确定刷新日志到磁盘中的频次；（可以有效减少IO性能损耗）

 ddl、dml -- commit -- 语句信息 -- 内存中 - 系统决定 - 日志文件中

参数官方资料链接：https://dev.mysql.com/doc/refman/8.0/en/replication-options-binary-log.html

> binlog_format  -- 影响binlog日志记录的格式
>
> binlog_format=statement  语句格式记录（SBR），以语句形式记录binlog信息，会将DML语句信息可读方式记录：但是一些函数（比如时间、随机函数会有问题）
>
> binlog_format=row ，行格式记录（RBR）， 推荐，以数据行的内容记录binlog信息，会将DML语句信息编码方式记录；
>
> binlog_format=mixed，混合记录方式， 数据库服务自动识别语句信息，判断是否使用RBR 或是使用 SBR

### 3.3.4 日志信息清理方法

日志信息自动清理

```SQL
mysql> show variables like '%expire%';
+--------------------------------+---------+
| Variable_name                  | Value   |
+--------------------------------+---------+
| binlog_expire_logs_seconds     | 2592000 |   -- 多少秒后，将历史binlog日志清理
| expire_logs_days               | 0       |
+--------------------------------+---------+
```

日志信息手工清理

```Bash
mysql> purge binary logs to 'mysql-bin.010' -- 删除到指定日志文件前结束

mysql> PURGE BINARY LOGS BEFORE '2019-04-02 22:46:26'; -- 可以基于日志时间点信息进行日志清理
```

清理 binlog 日志原则：1-2两个全备周期前的 binlog 日志做清理。

周一       周二             周日

数据信息   数据信息  ....  数据信息   全备  

binlog001  binlog002        binlog07   backup.sql   1个备份周期

周一       周二             周日

数据信息   数据信息  ....  数据信息   全备  

binlog008  binlog009        binlog14   backup.sql   2个备份周期

## 3.4. 慢查询日志

慢日志主要是用于以文本形式记录数据库服务运行过程中，执行过程较慢的语句；

利用慢日志信息生成的信息，可以在日常巡检过程中，通过日志定位SQL语句性能问题；

```SQL
select @@slow_query_log;  
-- 确认慢查询日志是否处于开启状态
select @@slow_query_log_file;
-- 指定慢查询日志保存路径，以及设置日志名称信息
select @@long_query_time;
-- 定义超过指定时间信息慢查询语句才会记录日志
select @@log_queries_not_using_indexes;
-- 将没有走索引的语句信息进行记录

Time                 Id Command    Argument
Time: 2024-06-27T00:44:58.898492Z
User@Host: root[root] @ localhost []  Id:     9
Query_time: 1.042358  Lock_time: 0.000125 Rows_sent: 1000000  Rows_examined: 1000000
use oldboy;
SET timestamp=1719449097;
select * from t100w;
```

慢查询日志统计分析

```Bash
mysqldumpslow -s c -t 3 /data/3306/log/xiaoQ_slowlog.log
```
