# 1. CA 机构

1. 自建 CA

```shell
openssl genrsa -out ca.key 4096
```

2. 生成 CA 证书

```shell
openssl req -x509 -new -nodes -sha512 -days 3650 \
 -subj "/C=CN/ST=Hebei/L=XiongAn/O=zongbu/OU=operation/CN=Zongbu Root CA" \
 -key ca.key \
 -out ca.crt
```

# 2. 服务器证书

1. 生成服务器私钥 key

```shell
openssl genrsa -out yourdomain.com.key 4096
```

2. 由服务器私钥创建服务器待签名证书
```shell
openssl req -sha512 -new \
    -subj "/C=CN/ST=Hebei/L=XiongAn/O=zigongsi/OU=operation/CN=yourdomain.com" \
    -key yourdomain.com.key \
    -out yourdomain.com.csr
```

3. 生成 x509 v3 扩展文件。
```shell
cat > v3.ext <<-EOF
authorityKeyIdentifier=keyid,issuer
basicConstraints=CA:FALSE
keyUsage = digitalSignature, nonRepudiation, keyEncipherment, dataEncipherment
extendedKeyUsage = serverAuth
subjectAltName = @alt_names

[alt_names]
DNS.1=*.yourdomain.com
DNS.2=yourdomain.com
EOF
```

3. 利用 CA 对服务器证书签名

```shell
openssl x509 -req -sha512 -days 36500 \
    -extfile v3.ext \
    -CA ca.crt -CAkey ca.key -CAcreateserial \
    -in yourdomain.com.csr \
    -out yourdomain.com.crt
```

# 3. 接入 nginx

把 server.crt 和 server.key 上传到 nginx

```shell
"server": {
    "listen": "443 ssl",
    "server_name": "xx.com",
    "ssl_certificate": "/data/server.crt",
    "ssl_certificate_key": "/data/server.key"
}
```

# 4. 去除不安全认证

去除不安全的情况

1. 浏览器导入 CA 证书，ca.crt
2. 若上一步不行，操作系统导入 CA 证书，ca.crt
