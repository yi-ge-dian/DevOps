wal 文件介绍

```text
00000018 00000000 00000028
\________/ \______/ \______/
  Timeline   高32位   低32位
    (8位)    (8位)    (8位)

完整格式：{TimelineID}{LogID}{SegID}
总共 24 个十六进制字符（24 hex digits），每个部分 8 个十六进制字符（8 hex digits），
\
部分	值（十六进制）	十进制值	含义
TimelineID	00000018	24	时间线 ID
LogID（高32位）	00000000	0	WAL 逻辑日志文件号的高位部分
SegID（低32位）	00000028	40	WAL 逻辑日志文件号的低位部分
```

完整的逻辑文件序号：LogID (32位) + SegID (32位) = 0x0000000000000028 = 40

查看主从复制状态

```sql
SELECT client_addr, state, sent_lsn, write_lsn, flush_lsn, replay_lsn FROM pg_stat_replication;
```

- client_addr：连接到主库的从库的 IP 地址。
- state：复制状态，"streaming" 表示正在流式复制。
- sent_lsn：主库已经发送给从库的最新 WAL 日志位置。
- write_lsn：从库已经写入磁盘的最新 WAL 日志位置。
- flush_lsn：从库已经刷新到磁盘的最新 WAL 日志位置。
- replay_lsn：从库已经应用到数据库的最新 WAL 日志位置。

如果想清除数据，应该以 replay_lsn 为准。

查看相应的 wal 文件：

```sql
SELECT pg_walfile_name('11F/FBB92FF0');
```

清理

```sql
pg_archivecleanup -d /data/5432/archive  000000190000011F000000DB
```
